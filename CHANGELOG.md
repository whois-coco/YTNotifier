# Changelog

## v0.8.1

### ログレベル・分類の見直し

- `CheckStarted` / `CheckCompleted`（チェック開始/完了）を SYSTEM → INFO に変更
- `NetworkRestored`（接続回復）を INFO → SYSTEM に変更
- `NetworkDisconnected`（接続切断）を WARNING → ERROR に変更
- `QuotaExceeded`（クォータ上限到達）を WARNING → ERROR に変更
- `ApiKeyChanged`（APIキー変更）を DEBUG → INFO に変更
- `SettingBackupExported` / `SettingBackupImported`（バックアップ）を INFO → SYSTEM に変更
- `QuotaAutoIntervalAdjusted`（クォータ超過による間隔自動調整）を DEBUG → WARNING に変更
- `ApiKeyMigrated` / `BackupSaved` を INFO → DEBUG に変更
- `QuotaExceededOnSave` を ERROR → WARNING に変更
- `QuotaRiskOverLimit`（未使用）を削除

### ログフィルターの動作変更

- ログレベル「INFO」設定時に WARNING も表示されるよう修正（従来は INFO と ERROR のみ）

### トレイアイコン操作のログ追加

- システムトレイメニューの「ウィンドウを開く」「今すぐチェック」「監視開始」「監視停止」の操作を DEBUG ログに記録するよう追加

### クォータ85%警告の修正

- チャンネル追加時のクォータ85%警告を、画面内テキスト表示（ユーザーが目にする前に消える不具合あり）から確認ダイアログに変更

### クォータ超過ダイアログのメッセージ変更

- チャンネル追加時のクォータ超過ダイアログに「追加すると監視間隔が {X}分 に変更されます」を表示するよう変更

## v1.8.1

### 監視モードの全面再設計（種別別スロット化）
- チャンネル詳細設定から「監視モード」のグローバル選択を廃止し、**動画・Short・ライブ配信** の3タブを常時表示
- 各タブが独立した `SlotMode`（通常/低頻度/時間指定）を持ち、種別ごとに異なる監視頻度を設定可能に
- タブの「この設定を有効にする」が `NotifyVideo/Short/Live` と双方向同期。タブ有効化時はデフォルトで「通常」モードに設定
- 「通常」モードでも個別の監視間隔（3〜60分、0=グローバル設定）を選択可能に
- 詳細設定を一度も開いていないチャンネルも、起動時に旧データから自動で新形式（3スロット）へ一括移行（`SettingsService.LoadChannels()` 内）
- チャンネル一覧の種別アイコン色が種別ごとの実効モードを反映（水色=通常／黄色=低頻度／黄緑=時間指定）
- API 使用量(%) の試算もスロット単位の実際のモードに基づいて計算するよう修正

### タスクバー点滅機能
- 設定→通知に「タスクバー点滅」トグルを追加。通知時に `FlashWindowEx` でタスクバーを点滅
- ミュートボタン ON/OFF 時に他の通知設定と同様に退避・復元

### 動作ログの改善
- 動作ログウィンドウが当日の全ログを時刻付きで表示するように変更（`LoggerService.TodayEntries`）
- 「クリア」ボタンを押すと表示のみクリアされ、以後は新規メッセージのみ表示されるよう修正（ログファイルは保持）

### 設定画面の変更
- ログフィルタ（レベル・自動削除・保持期間）を「ログフィルタ」専用タブから「表示」タブ内に統合
- 「表示」タブ内でダークモード・カテゴリなし表示をグループ化
- 「通知」タブ内で通知関連設定（通知テストを除く）をグループ化
- 設定内の全ドロップダウンの幅をコンテンツサイズに合わせて自動調整（右寄せ）
- サブナビゲーションの並び順を「表示・通知・動作・ABOUT」に変更し、「基本」タブを「動作」に改名
- 起動時に毎回バックアップが実行されてしまう不具合を修正（コンパクトモードトグルの初期化処理が原因）

### ドキュメント構成の変更
- `README.md` をインストール方法・初期設定・使い方・免責事項のみに整理
- 機能の詳細説明を `FUNCTION.md` に分離
- 変更履歴は本ファイル `CHANGELOG.md` に継続記載

## v1.8.0

### 新機能

#### 動画種別判定の全面刷新
- 4フェーズ判定ロジックを実装
  - フェーズ1: `liveBroadcastContent` + `uploadStatus` によるライブ/プレミア判定
  - フェーズ2: `liveStreamingDetails` + `publishedAt ≒ actualStartTime` によるアーカイブ判定
  - フェーズ3: `duration > 180秒` による通常動画の早期確定
  - フェーズ4: HEAD リクエスト + UUSH プレイリストによる Shorts 判定
- `VideoKind.Premiere`（プレミア公開）を新規追加
- プレミア公開後の動画化・ライブアーカイブ化では再通知しない仕様

#### トースト通知スタイル
- 設定→監視に「通知スタイル」ドロップダウンを追加
  - **デフォルト通知**: チャンネルアイコン + チャンネル名 + タイトル
  - **サムネイル通知**: 動画サムネイルをヒーロー画像として大きく表示
- サムネイルはローカルキャッシュ経由で表示（`thumbcache` フォルダ、7日で自動削除）

#### プレミア/ライブ待機所通知
- チャンネル詳細設定に「プレミア/ライブ待機所通知」トグルを追加
  - **ON**: `upcoming`（待機所）状態で即通知
  - **OFF**: `live` ステータスになった時に通知
- 設定→監視にグローバル一括 ON/OFF を追加（ON/OFF 変更時に全チャンネルへ適用）

#### オフライン検知
- ネットワーク切断時に自動で監視を停止
- サイドバーアイコンが赤色の WiFi オフアイコンに変化
- チャンネルリストが非表示になり「インターネット接続がありません。」を表示
- 5秒ごとの Ping ポーリングで疎通を確認、回復時に自動で監視を再開
- API チェック失敗時にも即時オフライン検知をトリガー

#### トピックチャンネル対応
- チャンネル追加時に `channels.list` から正確な `UploadsPlaylistId` を取得・保存
- UC → UU 変換に頼らずトピックチャンネルの動画も確実に検知

#### 通知フィルターの改善
- 通知対象外の種別をスキップして次の動画を探す動作に変更
  - 例: 動画のみ ON で最新がライブの場合、ライブをスキップして通常動画を通知

### 設定画面の変更

#### 設定→監視
- 通知音とデスクトップ通知の表示順を入れ替え
- プレミア/ライブ待機所通知の一括設定を追加
- 通知スタイルの選択を追加

#### 詳細設定（旧: 上級設定）
- 名称を「上級設定」から「詳細設定」に変更（コンテキストメニューも同様）
- 時間指定モードの前後幅に ±1分・±3分 を追加
- 時間指定モードの監視間隔に 1分 を追加
- 「プレミア/ライブ待機所通知」トグルを追加

### UI の改善
- チャンネル追加ウィンドウの貼り付けボタンを入力欄内側に配置
- コンパクトモード + 編集モード ON 時に各チャンネル行右端に削除ボタン（赤）を表示
- ウィンドウ最小高さを 500px に変更

### バックアップ
- `Sounds` フォルダをフォルダごとバックアップ対象に追加
- 復元時に `YTNotifier.exe` と同じディレクトリの `Sounds\` へ展開

### API クォータ管理
- 使用量リセット基準を YouTube のクォータリセット時刻（日本時間 16:00）に変更

### 内部リファクタリング
- `NotificationService.cs` を新規作成（トースト通知・サウンド再生を MonitorService から分離）
- `ImageCacheService.cs` を新規作成（アイコン・サムネイルキャッシュ管理を分離）
- `TrayIconService.cs` を新規作成（トレイアイコン管理を App.xaml.cs から分離）
- `UpdateChannelSilent()` を追加（監視系の状態更新でダーティフラグを立てない）
- 設定読み込み時のトグル/ComboBox イベント誤発火を修正（`_loadingSettings` フラグ）
- ログフィルター設定が保存されない問題を修正
