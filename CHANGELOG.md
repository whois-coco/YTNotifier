# Changelog

## v1.8.0

### 新機能

#### 動画種別判定の全面刷新
- 4フェーズ判定ロジックを実装
  - フェーズ1: `liveBroadcastContent` + `uploadStatus` によるライブ/プレミア判定
  - フェーズ2: `liveStreamingDetails` + `publishedAt ≒ actualStartTime` によるアーカイブ判定
  - フェーズ3: `duration > 180秒` による通常動画の早期確定
  - フェーズ4: HEAD リクエスト + UUSH プレイリストによる Shorts 判定
- `VideoKind.Premiere`（プレミア公開）を新規追加
- プレミア公開後の動画化・ライブアーカイブ化では再通知しない仕様

#### トースト通知スタイル
- 設定→監視に「通知スタイル」ドロップダウンを追加
  - **デフォルト通知**: チャンネルアイコン + チャンネル名 + タイトル
  - **サムネイル通知**: 動画サムネイルをヒーロー画像として大きく表示
- サムネイルはローカルキャッシュ経由で表示（`thumbcache` フォルダ、7日で自動削除）

#### プレミア/ライブ待機所通知
- チャンネル詳細設定に「プレミア/ライブ待機所通知」トグルを追加
  - **ON**: `upcoming`（待機所）状態で即通知
  - **OFF**: `live` ステータスになった時に通知
- 設定→監視にグローバル一括 ON/OFF を追加（ON/OFF 変更時に全チャンネルへ適用）

#### オフライン検知
- ネットワーク切断時に自動で監視を停止
- サイドバーアイコンが赤色の WiFi オフアイコンに変化
- チャンネルリストが非表示になり「インターネット接続がありません。」を表示
- 5秒ごとの Ping ポーリングで疎通を確認、回復時に自動で監視を再開
- API チェック失敗時にも即時オフライン検知をトリガー

#### トピックチャンネル対応
- チャンネル追加時に `channels.list` から正確な `UploadsPlaylistId` を取得・保存
- UC → UU 変換に頼らずトピックチャンネルの動画も確実に検知

#### 通知フィルターの改善
- 通知対象外の種別をスキップして次の動画を探す動作に変更
  - 例: 動画のみ ON で最新がライブの場合、ライブをスキップして通常動画を通知

### 設定画面の変更

#### 設定→監視
- 通知音とデスクトップ通知の表示順を入れ替え
- プレミア/ライブ待機所通知の一括設定を追加
- 通知スタイルの選択を追加

#### 詳細設定（旧: 上級設定）
- 名称を「上級設定」から「詳細設定」に変更（コンテキストメニューも同様）
- 時間指定モードの前後幅に ±1分・±3分 を追加
- 時間指定モードの監視間隔に 1分 を追加
- 「プレミア/ライブ待機所通知」トグルを追加

### UI の改善
- チャンネル追加ウィンドウの貼り付けボタンを入力欄内側に配置
- コンパクトモード + 編集モード ON 時に各チャンネル行右端に削除ボタン（赤）を表示
- ウィンドウ最小高さを 500px に変更

### バックアップ
- `Sounds` フォルダをフォルダごとバックアップ対象に追加
- 復元時に `YTNotifier.exe` と同じディレクトリの `Sounds\` へ展開

### API クォータ管理
- 使用量リセット基準を YouTube のクォータリセット時刻（日本時間 16:00）に変更

### 内部リファクタリング
- `NotificationService.cs` を新規作成（トースト通知・サウンド再生を MonitorService から分離）
- `ImageCacheService.cs` を新規作成（アイコン・サムネイルキャッシュ管理を分離）
- `TrayIconService.cs` を新規作成（トレイアイコン管理を App.xaml.cs から分離）
- `UpdateChannelSilent()` を追加（監視系の状態更新でダーティフラグを立てない）
- 設定読み込み時のトグル/ComboBox イベント誤発火を修正（`_loadingSettings` フラグ）
- ログフィルター設定が保存されない問題を修正
