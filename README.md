# YTNotifier v1.8.0 - YouTube新着動画通知システム

YouTube Data API v3 を利用し、指定チャンネルの新着動画を定期監視・通知する WPF 常駐アプリ。

---

## 動作環境

- Windows 10 / 11
- .NET 8.0 以上
- YouTube Data API v3 APIキー

---

## セットアップ

### 1. ビルド

```bash
cd YTNotifier
dotnet restore
dotnet build
```

### 2. 発行（Self-Contained）

```bash
dotnet publish -r win-x64 -p:PublishSingleFile=true --self-contained
```

### 3. リソースの準備

`Resources\` フォルダに以下のファイルを用意してください。

| ファイル | 用途 | 備考 |
|---|---|---|
| `app.ico` | ウィンドウ・トレイ通常時 | 必須 |
| `app_warn.ico` | トレイ停止時 | 必須 |
| `app.png` | トースト通知アイコン | 必須（.ico 不可） |
| `notification.wav` | カスタム通知音 | 任意・WAV形式 |

> `app.png` は `YTNotifier.csproj` に `<Content CopyToOutputDirectory="PreserveNewest">` で登録してください。

### カスタム通知音（任意）

`Resources\notification.wav` を配置すると通知音オン時に再生されます。ない場合は Windows のシステム通知音を使用します。

---

## YouTube API キーの取得

1. [Google Cloud Console](https://console.developers.google.com) を開く
2. プロジェクトを作成または選択
3. 「APIとサービス」→「ライブラリ」で **YouTube Data API v3** を有効化
4. 「認証情報」→「認証情報を作成」→「APIキー」を選択
5. アプリの「設定 → アプリ」画面に APIキーを貼り付けて保存

## 主要機能

### チャンネル管理
- チャンネルID / ハンドル名（`@name`） / URL で追加
- 追加前にプレビュー確認（チャンネル名・登録者数・アイコン）
- **ドラッグアンドドロップ**でリスト内の順序を変更
- **カテゴリ**でグループ化・折り畳み・ドラッグ移動対応
- チャンネルアイコンをクリックで最新動画をブラウザで開く
- 右クリックメニュー：NEWバッジを消す / 名称変更 / カテゴリ移動
- カテゴリ右クリック：全展開/閉じ / NEWバッジを全て消す / 名称変更 / 削除

### 通知種別フィルター
各チャンネルごとに **動画 / Short / ライブ** の通知種別を個別設定。
アイコンの色は監視モードと連動します。

| 監視モード | アイコン色 |
|---|---|
| 通常・低頻度 | 青 |
| 時間指定 | 緑 |

### 監視・通知
- API：`PlaylistItems.list`（1ユニット/チャンネル）+ `videos.list`（1ユニット）を使用
- `Activities.list`（100ユニット）は**使用しない**
- 複数の新着があっても通知は**最新1件のみ**（多重通知防止）
- Short判定：150秒以内、または `#Shorts`/`#Short` タグを含む
- ライブ判定：`liveStreamingDetails` フィールドが存在する動画
- **NEWバッジ**：新着検出時にチャンネル名横に表示
- Windowsトースト通知：チャンネルアイコン（丸型）＋チャンネル名＋動画タイトル
- **通知音**：`Resources\notification.wav` または Windows の `notify.wav` を25%音量で再生

### 上級設定（チャンネルごと）
チャンネル行の右クリックまたは編集モードからアクセス。

**監視モード：**

| モード | 内容 |
|---|---|
| 通常 | グローバルのチェック間隔で監視（個別間隔も設定可） |
| 低頻度 | 1〜24時間間隔で監視（あまり更新しないチャンネル向け） |
| 時間指定 | 設定1〜4の時間帯のみ集中監視（最大4スロット） |

**時間指定スロット（設定1〜4）：**
- 有効化チェックボックス（OFFの場合は設定項目をグレーアウト）
- 通知種別（動画 / Short / ライブ配信）
- 曜日指定（Sun〜Sat）
- 投稿時刻（時・分ドロップダウン）
- 前後の幅 / 監視間隔

> プレミア公開はAPIの仕様上「ライブ配信」として検知されます。

### バックアップ
- **手動バックアップ**：設定→アプリ→バックアップ（SaveFileDialog / デスクトップ初期）
- **手動復元**：設定→アプリ→復元（OpenFileDialog / デスクトップ初期）
- **自動バックアップ**：チャンネル・カテゴリ・設定変更後のアプリ終了時に `bkup\auto_backup.ytbk` へ保存（1世代）
- **自動復元**：起動時に以下を検知した場合、サイレントで自動復元
  - 設定ファイルが存在しない
  - ファイルサイズが0バイト
  - JSONパース失敗（破損）
  - チャンネルが0件（バックアップに1件以上ある場合）
- バックアップ形式：`.ytbk`（AES-256暗号化ZIP）
- ウィンドウ位置・サイズ変更はバックアップ対象外

### API使用量管理
- **推定使用量**：チャンネル数・間隔から計算（設定→監視）
- **本日の実使用量**：APIコール時にリアルタイム集計（設定→監視）
- 日付をまたぐと自動リセット
- 使用量バーは4段階でカラー表示：緑（0〜60%）/ 黄（61〜75%）/ オレンジ（76〜85%）/ 赤（86%以上）

### UI
- ライト / ダークモードをリアルタイム切替
- 折り畳み可能なサイドバー（状態を保存・復元）
- **コンパクトモード**：サイドバー強制折り畳み・アイコン20px・カテゴリ行高24px縮小
- 常に前面に表示（ピンボタン）
- **通知ミュート**：監視は継続したままデスクトップ通知と通知音を無効化
- チャンネル検索（リアルタイムフィルター）
- ウィンドウ最小縦サイズ：600px

### 動作ログ
- ログフィルター（設定→ログフィルタ）に従いUI・ファイル両方の出力を制御
- システムメッセージ（自動バックアップ・復元等）は常に出力
- 自動バックアップ実行時・自動復元実行時はログに記録

### システムトレイ
- 右クリックメニュー（ダークテーマ対応）：ウィンドウを開く / 今すぐチェック / 監視開始・停止 / 終了

## APIクォータ管理

YouTube Data API v3 の無料枠は **10,000ユニット/日**。

| 操作 | コスト |
|---|---|
| チャンネル情報取得 | 1ユニット |
| 新着チェック（PlaylistItems.list） | 1ユニット × チャンネル数 |
| 動画種別判定（videos.list） | 1ユニット × 新着数 |

**通常モードのみの目安：**

| 構成 | 推定消費量 |
|---|---|
| 10ch・5分間隔 | ≈ 5,760ユニット/日 |
| 10ch・10分間隔 | ≈ 2,880ユニット/日 |
| 28ch・3分間隔（全通常） | ≈ 26,880ユニット/日（超過）|
| 28ch・3分間隔（低頻度多用） | ≈ 3,000〜6,000ユニット/日 |

低頻度モード・時間指定モードを活用することでクォータを大幅に節約できます。

## 注意事項

- Windows 10 以降のトースト通知を使用します
- `app.png` はビルド出力ディレクトリにコピーされる必要があります（csprojに登録要）
- APIキー未設定の場合、チャンネル追加・監視機能は動作しません
- Self-Contained 発行時は `--self-contained` オプションを付けてください
- ウィンドウ最小サイズ：幅330px・高さ600px
