# YTNotifier

YouTube チャンネルの新着動画・ライブ配信・プレミア公開をリアルタイムで通知する Windows デスクトップアプリケーションです。

## 必要環境

- Windows 10 (build 17763) 以降
- .NET 8.0 ランタイム
- YouTube Data API v3 の API キー

---

## セットアップ

### 1. API キーの取得

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. プロジェクトを作成し、**YouTube Data API v3** を有効化
3. 認証情報から **API キー** を作成

### 2. アプリの初期設定

1. `YTNotifier.exe` を起動
2. API キーの入力画面が表示されるので取得したキーを入力
3. 設定→監視 から **デスクトップ通知** または **通知音** を ON にする
4. チャンネルを追加して監視を開始

---

## チャンネルの追加

サイドバーの **+** ボタンをクリックし、以下のいずれかの形式で入力します。

| 形式 | 例 |
|---|---|
| チャンネル URL | `https://www.youtube.com/@channelname` |
| ハンドル | `@channelname` |
| チャンネル ID | `UCxxxxxxxxxxxxxxxxxxxxxxxxx` |

クリップボードからの貼り付けボタン（入力欄内右側のアイコン）も利用できます。

---

## 通知の種類

YTNotifier は以下の種別を自動判定して通知します。

| 種別 | 説明 |
|---|---|
| **動画** | 通常の動画投稿 |
| **Short** | YouTube Shorts（3分以内の縦型動画） |
| **ライブ** | ライブ配信（配信中・アーカイブ） |
| **プレミア** | プレミア公開（公開前・公開中） |

### 判定ロジック

1. `liveBroadcastContent` + `uploadStatus` → ライブ配信 / プレミア公開
2. `liveStreamingDetails` の有無と `publishedAt ≒ actualStartTime` → アーカイブ判定
3. `duration > 180秒` → 通常動画（即確定）
4. HEAD リクエスト → Shorts / 通常動画

---

## 監視モード

チャンネルごとに3つの監視モードを設定できます（詳細設定から）。

| モード | 動作 |
|---|---|
| **通常** | グローバル設定のチェック間隔で定期チェック |
| **低頻度** | 指定間隔（60分〜）でチェック。更新頻度の低いチャンネル向け |
| **時間指定** | 指定した時刻の前後のみ集中的にチェック。最大4スロット・曜日指定可能 |

### 時間指定モードの例

- 投稿時刻: 20:00 / 前後の幅: ±5分 / 監視間隔: 1分
  → 19:55〜20:05 の間、1分ごとにチェック（計11回）

## 通知設定

### グローバル設定（設定→監視）

| 設定 | 説明 |
|---|---|
| 通知音 | 新着検知時にサウンドを再生（`Sounds\` フォルダの wav を使用） |
| デスクトップ通知 | Windows トースト通知を表示 |
| プレミア/ライブ待機所通知 | `upcoming`（待機所）段階で全チャンネルに通知（一括設定） |
| 通知スタイル | **デフォルト通知** または **サムネイル通知** を選択 |

### チャンネル個別設定（詳細設定）

| 設定 | 説明 |
|---|---|
| 通知種別 | 動画 / Short / ライブ配信 の通知を個別に ON/OFF |
| プレミア/ライブ待機所通知 | このチャンネルの待機所通知を個別に設定 |

### プレミア/ライブ待機所通知について

- **ON**: プレミア公開・ライブ配信の予約が作成された時点（`upcoming`）で通知
- **OFF**: 実際に `live` ステータスになった時に通知（デフォルト）

---

## 通知スタイル

### デフォルト通知
チャンネルアイコン（丸型）・チャンネル名・動画タイトルを表示するシンプルなスタイル。

### サムネイル通知
動画サムネイルをヒーロー画像として上部に大きく表示し、チャンネル名・種別・タイトルを表示するリッチなスタイル。

---

## 通知音のカスタマイズ

`YTNotifier.exe` と同じフォルダの `Sounds\` に以下のファイルを配置すると種別ごとに再生されます。

| ファイル名 | 再生タイミング |
|---|---|
| `video.wav` | 通常動画・プレミア公開 |
| `short.wav` | Shorts |
| `live.wav` | ライブ配信 |
| `test.wav` | 通知テストボタン押下時（優先） |

ファイルが存在しない場合はシステムサウンドにフォールバックします。

---

## バックアップと復元

設定→アプリ から操作できます。

- **バックアップ**: `config.json` / `channels.json` / `categories.json` / `api_key.dat` / `Sounds\` フォルダを `.ytbk`（AES-256 暗号化）で保存
- **復元**: `.ytbk` または `.zip` ファイルから設定を復元

---

## API クォータについて

YouTube Data API v3 の無料枠は **10,000 ユニット/日** です。

- リセット時刻: **毎日 16:00（日本時間）**
- チェック1回あたり: 約 2 ユニット（PlaylistItems.list + Videos.list）
- 設定→監視 の API 使用量(%) で現在の消費状況を確認できます

クォータを節約するには：
- チェック間隔を長くする
- 低頻度モードや時間指定モードを活用する
- 不要なチャンネルの監視を無効にする

---

## オフライン時の動作

ネットワーク切断を検知すると：

1. 監視を自動停止
2. サイドバーアイコンが赤色の WiFi オフアイコンに変化
3. チャンネルリストが「インターネット接続がありません。」に切り替わる

接続が回復すると自動で監視を再開します。

## データの保存場所

| データ | パス |
|---|---|
| 設定ファイル | `%AppData%\YTNotifier\conf\` |
| ログ | `%AppData%\YTNotifier\logs\` |
| アイコンキャッシュ | `%AppData%\YTNotifier\icons\` |
| サムネイルキャッシュ | `%AppData%\YTNotifier\thumbcache\`（7日で自動削除） |
| 通知音 | `{exeフォルダ}\Sounds\` |

## 開発者ツール（オプション）

`YTNotifier.Debug.dll` を `YTNotifier.exe` と同じフォルダに配置すると、設定→ABOUT に「開発者ツールを開く」ボタンが表示されます。

主な機能：
- チャンネルごとの API 実行と動画一覧表示
- 動画 URL / ID の種別判定テスト（判定根拠の詳細表示）
- 監視状態（NextCheckAt・LastCheckedAt 等）の一覧確認
- 設定ファイルの内容表示
- ログファイルの閲覧
- システム情報・API 使用量の確認
