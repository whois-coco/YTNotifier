# YTNotifier

YouTube チャンネルの新着動画・ライブ配信・プレミア公開をリアルタイムで通知する Windows デスクトップアプリケーションです。

機能の詳細は [FUNCTION.md](FUNCTION.md)、変更履歴は [CHANGELOG.md](CHANGELOG.md) を参照してください。

---

## 必要環境

- Windows 10 (build 17763) 以降
- .NET 8.0 ランタイム
- YouTube Data API v3 の API キー

---

## インストール方法

1. リリースされた `YTNotifier.exe`（または発行済みフォルダ）を任意の場所に配置します
2. `YTNotifier.exe` を実行します（インストーラ不要）

### Windows Defender SmartScreen の警告が表示された場合

初回起動時に「Windows によって PC が保護されました」という警告ダイアログが表示される場合があります。これはコード署名証明書のない実行ファイルに対して Windows が表示する警告です。

以下の手順でそのまま実行できます。

1. 警告ダイアログ内の「**詳細情報**」をクリックします
2. 「**実行**」ボタンが表示されるのでクリックします

または、ファイルのプロパティから事前にブロックを解除することもできます。

1. `YTNotifier.exe` を右クリックし「プロパティ」を開きます
2. 「全般」タブの下部に「**セキュリティ: このファイルは別のコンピューターから取得したものです。このコンピューターを保護するため、このファイルへのアクセスはブロックされる可能性があります。**」という文言と「**ブロックの解除**」チェックボックスが表示されます
3. チェックボックスをオンにして「OK」をクリックします
4. 以後は警告なしで起動できます

## 初期設定

### 1. API キーの取得

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. 画面上部のプロジェクト選択から新規プロジェクトを作成（既存プロジェクトでも可）
3. 左メニュー「APIとサービス」→「ライブラリ」を開き、**YouTube Data API v3** を検索して「有効にする」をクリック
4. 左メニュー「APIとサービス」→「認証情報」を開き、「認証情報を作成」→「APIキー」を選択
5. 発行された API キーをコピーしておく（後でアプリに入力します）
6. 必要に応じて「キーを制限」から API 制限を「YouTube Data API v3」のみに絞ると安全性が高まります

### 2. アプリの初回起動

1. `YTNotifier.exe` を起動すると、初回のみ **API キー入力画面** が表示されます
2. 手順1で取得した API キーを貼り付けて保存します
   - 後から変更したい場合は 設定→動作 の「API キー」欄から「変更」を選択することで再設定できます
3. 保存が完了するとメイン画面が表示されます（チャンネル一覧は空の状態です）

### 3. 推奨される初期設定

メイン画面右上のナビゲーションから「設定」タブを開き、以下を確認・設定します。

| タブ | 設定項目 | 説明 |
|---|---|---|
| 通知 | デスクトップ通知 | Windows のトースト通知を表示するか（ON 推奨） |
| 通知 | 通知音 | 新着検知時にサウンドを再生するか |
| 通知 | タスクバー点滅 | 通知時にタスクバーを点滅させるか |
| 通知 | プレミア/ライブ待機所通知 | 待機所（upcoming）の段階で通知するか、配信開始後に通知するか |
| 通知 | 通知スタイル | デフォルト通知 / サムネイル通知 を選択 |
| 表示 | ダークモード | 配色をダークテーマに切り替え |
| 表示 | カテゴリなし表示 | チャンネルをカテゴリ分けせずフラット表示するか |
| 動作 | チェック間隔 | グローバルなチェック間隔（通常モードの既定値） |
| 動作 | 起動時に自動実行 | Windows 起動時にアプリを自動起動するか |
| 動作 | トレイに最小化 | ウィンドウを閉じた時にタスクトレイに格納するか |

設定はすべて即時保存されるため、「保存」ボタンの操作は不要です。各設定の詳細は [FUNCTION.md](FUNCTION.md) を参照してください。

### 4. 最初のチャンネルを登録する

「使い方」の「チャンネルを追加する」を参照し、最低1つチャンネルを登録してから監視を開始してください。

## 使い方

### チャンネルを追加する

1. サイドバーの **+**（チャンネル追加）ボタンをクリックするとチャンネル追加ダイアログが開きます
2. 入力欄に以下のいずれかの形式で入力します

   | 形式 | 例 |
   |---|---|
   | チャンネル URL | `https://www.youtube.com/@channelname` |
   | ハンドル | `@channelname` |
   | チャンネル ID | `UCxxxxxxxxxxxxxxxxxxxxxxxxx` |

3. 入力欄右側の貼り付けアイコンをクリックすると、クリップボードの内容をそのまま貼り付けられます
4. 「プレビュー」ボタン（または Enter キー）を押すと、チャンネル名・アイコンが表示され入力内容を確認できます
5. 必要であれば登録先の「カテゴリ」を選択します（カテゴリなし表示モードが ON の場合は選択不可）
6. 「追加」ボタンで登録が完了します
7. ダイアログ内の「連続して追加」チェックボックスを ON にしておくと、追加後も入力欄がリセットされ、ダイアログを閉じずに次のチャンネルを続けて登録できます
8. 1日あたりの API クォータを超過する見込みがある場合は、追加前に警告（85%超過時）または確認ダイアログ（超過時）が表示されます

### チャンネルを整理する（カテゴリ・並び替え）

- チャンネル一覧右上の「カテゴリを作成」からカテゴリを新規作成できます
- チャンネル行・カテゴリ行はドラッグ＆ドロップで並び替え可能です（カテゴリ間の移動も可能）
- カテゴリ見出しをクリックすると展開/折り畳みができます
- チャンネル行を右クリックすると以下の操作ができます
  - NEW バッジを消す
  - 名称を変更
  - 新規カテゴリを作成して移動
  - 既存のカテゴリへ移動 / 未分類に戻す
  - 詳細設定を開く
- サイドバーの「編集モード」を ON にすると、各チャンネル行の右端に削除ボタン（赤）が表示され、チャンネルの削除や並び替えがしやすくなります
- 設定→表示 の「カテゴリなし表示」を ON にすると、カテゴリ分けせず全チャンネルをフラットな一覧として表示します（OFF に戻すと元のカテゴリ・順序が復元されます）

### 監視を開始する

- サイドバーの監視トグルボタンをクリックすると、監視の開始/停止を切り替えられます
- サイドバーの手動チェックボタンをクリックすると、全チャンネルを即時チェックします（通常の監視間隔を待たずに確認したい場合に使用）
- 監視中はサイドバーのステータスバッジが「監視中」、停止中は「停止中」と表示されます
- ネットワーク切断を検知すると自動的に監視が停止し、回復すると自動的に再開します（詳細は [FUNCTION.md](FUNCTION.md) の「オフライン時の動作」を参照）

### チャンネルごとの詳細設定を変更する

チャンネル行をクリック、または右クリックメニューから「詳細設定」を選ぶと詳細設定ウィンドウが開きます。

1. ウィンドウ上部で **動画 / Short / ライブ配信** の3つのタブが表示され、種別ごとに個別の監視設定が可能です
2. 各タブの「この設定を有効にする」チェックボックスで、その種別の通知 ON/OFF を切り替えます（チャンネル一覧の種別トグルとも連動）
3. 有効化すると監視モードの選択肢が表示されます
   - **通常**：グローバルのチェック間隔、または個別に指定した間隔でチェック
   - **低頻度**：60分〜24時間など長めの間隔でチェック（更新頻度の低いチャンネル向け）
   - **時間指定**：曜日・投稿予定時刻・監視時間幅（デフォルト15分）・チェック間隔を指定し、その時間帯のみ高頻度にチェック。曜日はデフォルトで全曜日ON（最低1曜日は必ず有効）
4. 「プレミア/ライブ待機所通知」をこのチャンネルだけ個別にON/OFFすることもできます
5. ウィンドウ下部の API 使用量バーで、この設定による消費クォータの見込みを確認できます
6. 「保存」をクリックすると設定が即座に反映されます

### 通知を確認する

- 新着動画・ライブ配信・プレミア公開を検知すると、Windows のトースト通知が表示されます（設定に応じて通知音・タスクバー点滅も発生）
- トースト通知をクリックすると、該当する動画ページがブラウザで開きます
- 通知スタイルは「デフォルト通知」（チャンネルアイコン+タイトル）と「サムネイル通知」（動画サムネイルを大きく表示）から選択できます

### 通知を一時的に止める（ミュート）

サイドバーのミュートボタンを ON にすると、デスクトップ通知・通知音・タスクバー点滅を一括で OFF にします。OFF に戻すと、ミュート前の設定状態がそれぞれ個別に復元されます。

### 動作ログを確認する

サイドバーから「動作ログ」を開くと、当日発生したログを時刻付きで一覧表示します。「クリア」ボタンで表示をクリアでき、以後は新規メッセージのみが表示されます（ログファイル自体は削除されません）。

### バックアップを取る・復元する

設定→動作 から、設定ファイル一式・通知音フォルダを暗号化された `.ytbk` ファイルとして保存できます。同じ画面から `.ytbk` / `.zip` ファイルを指定して復元することも可能です。

### その他の操作

- **コンパクトモード**：サイドバー下部のボタンから切替。ウィンドウを最小幅で表示するモード
- **常に前面に表示（ピン留め）**：サイドバー下部のボタンから切替。他のウィンドウより手前に固定表示
- **通知テスト**：実際の新着検知を待たずに、現在の通知スタイル・通知音設定でテスト通知を送信（設定→通知）

機能ごとのより詳しい説明は [FUNCTION.md](FUNCTION.md) を参照してください。
