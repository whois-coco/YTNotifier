# 機能詳細

YTNotifier の各機能について説明します。インストール方法・初期設定は [README.md](README.md)、変更履歴は [CHANGELOG.md](CHANGELOG.md) を参照してください。

---

## チャンネルの追加

サイドバーの **+** ボタンをクリックし、以下のいずれかの形式で入力します。

| 形式 | 例 |
|---|---|
| チャンネル URL | `https://www.youtube.com/@channelname` |
| ハンドル | `@channelname` |
| チャンネル ID | `UCxxxxxxxxxxxxxxxxxxxxxxxxx` |

クリップボードからの貼り付けボタン（入力欄内右側のアイコン）も利用できます。

## 通知の種類

YTNotifier は以下の種別を自動判定して通知します。

| 種別 | 説明 |
|---|---|
| **動画** | 通常の動画投稿 |
| **Short** | YouTube Shorts（3分以内の縦型動画） |
| **ライブ** | ライブ配信（配信中・アーカイブ） |
| **プレミア** | プレミア公開（公開前・公開中） |

### 判定ロジック

1. `liveBroadcastContent` + `uploadStatus` → ライブ配信 / プレミア公開
2. `liveStreamingDetails` の有無と `publishedAt ≒ actualStartTime` → アーカイブ判定
3. `duration > 180秒` → 通常動画（即確定）
4. HEAD リクエスト → Shorts / 通常動画

## 監視モード

チャンネルの詳細設定では **動画・Short・ライブ配信** の3つのタブが常に表示され、種別ごとに独立した監視モードを設定できます。

| モード | 動作 |
|---|---|
| **通常** | グローバル設定のチェック間隔（または個別指定の間隔）で定期チェック |
| **低頻度** | 指定間隔（60分〜）でチェック。更新頻度の低い種別向け |
| **時間指定** | 指定した時刻の前後のみ集中的にチェック。曜日指定可能 |

各タブの「この設定を有効にする」は、チャンネル一覧の種別トグル（動画/Short/ライブ）と連動しています。タブを有効化すると監視モードは自動的に「通常」になります。

チャンネル一覧の種別アイコンの色は、その種別の実効モードを示します。

| 色 | モード |
|---|---|
| 水色 | 通常 |
| 黄色 | 低頻度 |
| 黄緑 | 時間指定 |

### 時間指定モードの例

- 投稿時刻: 20:00 / 投稿監視時間: 10分 / 監視間隔: 1分
  → 20:00〜20:10 の間、1分ごとにチェック（計10回）

## 通知設定

### グローバル設定（設定→通知）

| 設定 | 説明 |
|---|---|
| 通知音 | 新着検知時にサウンドを再生（`Sounds\` フォルダの wav を使用） |
| タスクバー点滅 | 通知時にタスクバーを点滅させる（ON/OFF） |
| デスクトップ通知 | Windows トースト通知を表示 |
| プレミア/ライブ待機所通知 | `upcoming`（待機所）段階で全チャンネルに通知（一括設定） |
| 通知スタイル | **デフォルト通知** または **サムネイル通知** を選択 |

### チャンネル個別設定（詳細設定）

| 設定 | 説明 |
|---|---|
| 通知種別 | 動画 / Short / ライブ配信 の通知を個別に ON/OFF（タブの有効/無効と連動） |
| プレミア/ライブ待機所通知 | このチャンネルの待機所通知を個別に設定 |

### プレミア/ライブ待機所通知について

- **ON**: プレミア公開・ライブ配信の予約が作成された時点（`upcoming`）で通知
- **OFF**: 実際に `live` ステータスになった時に通知（デフォルト）

### ミュートボタン

サイドバーのミュートボタンを ON にすると、デスクトップ通知・通知音・タスクバー点滅を一時的にすべて OFF にします。OFF に戻すと、ミュート前の各設定状態を個別に復元します。

## 通知スタイル

### デフォルト通知
チャンネルアイコン（丸型）・チャンネル名・動画タイトルを表示するシンプルなスタイル。

### サムネイル通知
動画サムネイルをヒーロー画像として上部に大きく表示し、チャンネル名・種別・タイトルを表示するリッチなスタイル。

---

## 通知音のカスタマイズ

`YTNotifier.exe` と同じフォルダの `Sounds\` に以下のファイルを配置すると種別ごとに再生されます。

| ファイル名 | 再生タイミング |
|---|---|
| `video.wav` | 通常動画・プレミア公開 |
| `short.wav` | Shorts |
| `live.wav` | ライブ配信 |
| `test.wav` | 通知テストボタン押下時（優先） |

ファイルが存在しない場合はシステムサウンドにフォールバックします。

## バックアップと復元

設定→動作 から操作できます。

- **バックアップ**: `config.json` / `channels.json` / `categories.json` / `api_key.dat` / `Sounds\` フォルダを `.ytbk`（AES-256 暗号化）で保存
- **復元**: `.ytbk` または `.zip` ファイルから設定を復元

## API クォータについて

YouTube Data API v3 の無料枠は **10,000 ユニット/日** です。

- リセット時刻: **毎日 16:00（日本時間・夏時間）/ 17:00（日本時間・冬時間）**
- チェック1回あたり: 約 2 ユニット（PlaylistItems.list + Videos.list）
- 設定→動作 の下部にある「API使用量(%)」バーで推定消費量と本日の実使用量を確認できます

クォータを節約するには：
- チェック間隔を長くする
- 低頻度モードや時間指定モードを活用する
- 不要なチャンネルの監視を無効にする

## オフライン時の動作

ネットワーク切断を検知すると：

1. 監視を自動停止
2. サイドバーアイコンが赤色の WiFi オフアイコンに変化
3. チャンネルリストが「インターネット接続がありません。」に切り替わる

接続が回復すると自動で監視を再開します。

---

## 動作ログ

サイドバーから「動作ログ」を開くと、当日に発生したログを時刻付きで一覧表示します。「クリア」ボタンで表示をクリアでき、以後は新規メッセージのみが表示されます（ログファイル自体は削除されません）。ログレベルや自動削除・保持期間は 設定→表示 から変更できます。

## データの保存場所

| データ | パス |
|---|---|
| 設定ファイル | `%AppData%\YTNotifier\conf\` |
| ログ | `%AppData%\YTNotifier\logs\` |
| アイコンキャッシュ | `%AppData%\YTNotifier\icons\` |
| サムネイルキャッシュ | `%AppData%\YTNotifier\thumbcache\`（7日で自動削除） |
| 通知音 | `{exeフォルダ}\Sounds\` |
